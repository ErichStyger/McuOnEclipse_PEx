Component Wait
Wait
Component Level: High
Category: SW-User Components
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • Wait10Cycles - Wait for 10 CPU cycles.

      ANSIC prototype: void Wait10Cycles(void)

  • Wait100Cycles - Wait for 100 CPU cycles.

      ANSIC prototype: void Wait100Cycles(void)

  • WaitCycles - Wait for a specified number of CPU cycles.

      ANSIC prototype: void WaitCycles(uint32_t cycles)
    • cycles:uint32_t - The number of cycles to wait.

  • WaitLongCycles - Wait for a specified number of CPU cycles (32bit data type).

      ANSIC prototype: void WaitLongCycles(uint32_t cycles)
    • cycles:uint32_t - The number of cycles to wait.

  • Waitms - Wait for a specified time in milliseconds.

      ANSIC prototype: void Waitms(uint32_t ms)
    • ms:uint32_t - How many milliseconds the function has to wait

  • Waitus - Wait for a specified time in microseconds.

      ANSIC prototype: void Waitus(uint32_t us)
    • us:uint32_t - How many microseconds the function has to wait

  • Waitns - Wait for a specified time in nano seconds.

      ANSIC prototype: void Waitns(uint32_t ns)
    • ns:uint32_t - How many ns the function has to wait

  • WaitOSms - If an RTOS is enabled, this routine will use a non-blocking wait method. Otherwise it will do a busy/blocking wait.

      ANSIC prototype: void WaitOSms(void)

  • Init - Driver initialization routine.

      ANSIC prototype: void Init(void)

  • Deinit - Driver de-initialization routine

      ANSIC prototype: void Deinit(void)


Note: Some methods can be implemented as macros.
