Component GenericTimeDate
Generic Software Time and Date component
Component Level: High
Category: SW-User Components
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • AddTick - Increments the tick counter of the software RTC. Needs to be called periodically by the application to increase the time tick count.

      ANSIC prototype: void AddTick(void)

  • AddTicks - Same as AddTick(), but multiple ticks can be added in one step.

      ANSIC prototype: void AddTicks(uint16_t nofTicks)
    • nofTicks:uint16_t - Number of ticks to be added

  • CalculateDayOfWeek - Returns the day of the week, Sunday starting as zero. Monday is 1, ...

      ANSIC prototype: uint8_t CalculateDayOfWeek(uint16_t Year, uint8_t Month, uint8_t Day)
    • Year:uint16_t - Years (16-bit unsigned integer)
    • Month:uint8_t - Months (8-bit unsigned integer)
    • Day:uint8_t - Days (8-bit unsigned integer)
    • Return value:uint8_t - Error code

  • SetTime - This method sets a new actual time of the software RTC.

      ANSIC prototype: uint8_t SetTime(uint8_t Hour, uint8_t Min, uint8_t Sec, uint8_t Sec100)
    • Hour:uint8_t - Hours (0 - 23)
    • Min:uint8_t - Minutes (0 - 59)
    • Sec:uint8_t - Seconds (0 - 59)
    • Sec100:uint8_t - Hundredth of seconds (0 - 99)
    • Return value:uint8_t - Error code

  • GetTime - This method returns current time from the software RTC.

      ANSIC prototype: uint8_t GetTime(TIMEREC *time)
    • time: Pointer to TIMEREC - Pointer to the structure TIMEREC. It contains actual number of hours, minutes, seconds and hundredth of seconds.
    • Return value:uint8_t - Error code

  • SetDate - This method sets a new actual date of the software RTC.

  • UnixSecondsToTimeDateCustom - Transforms a given time/date into the Unix time stamp, with the number of seconds from a starting date

      ANSIC prototype: void UnixSecondsToTimeDateCustom(int32_t seconds, int8_t offset_hours, TIMEREC *time, DATEREC *date, uint16_t baseYear)
    • seconds:int32_t - Unix time stamp in seconds after the given base year and base month
    • offset_hours:int8_t - Optional time zone offset, use 0 for no offset
    • time: Pointer to TIMEREC - Pointer to TIMEREC struct where the result is stored. Can be NULL.
    • date: Pointer to DATEREC - Pointer to DATEREC struct where the result is stored. Can be NULL.
    • baseYear:uint16_t - Base year from which is used as base to count the seconds, in YYYY format.

  • UnixSecondsToTimeDate - Transforms a given time/date into the Unix time stamp, with the number of seconds after 1-Jan-1970

      ANSIC prototype: void UnixSecondsToTimeDate(int32_t seconds, int8_t offset_hours, TIMEREC *time, DATEREC *date)
    • seconds:int32_t - Unix time stamp in seconds after the given base year and base month
    • offset_hours:int8_t - Optional time zone offset, use 0 for no offset
    • time: Pointer to TIMEREC - Pointer to TIMEREC struct where the result is stored. Can be NULL.
    • date: Pointer to DATEREC - Pointer to DATEREC struct where the result is stored. Can be NULL.

  • TimeDateToUnixSecondsCustom - Returns for a given time/date the corresponding UNIX time stamp with a custom base date.

      ANSIC prototype: int32_t TimeDateToUnixSecondsCustom(TIMEREC *time, DATEREC *date, int8_t offset_hours, uint16_t baseYear)
    • time: Pointer to TIMEREC - Pointer to TIMEREC which holds the time to convert
    • date: Pointer to DATEREC - Pointer to DATEREC with the date to convert
    • offset_hours:int8_t - Optional number of offset hours to adjust for the time zone. Use 0 for no adjustment.
    • baseYear:uint16_t - Base year from which is used as base to count the seconds, in YYYY format
    • Return value:int32_t - number of seconds after given base month/year

  • TimeDateToUnixSeconds - Returns for a given time/date the corresponding UNIX time stamp, starting at 1-Jan-1970.

      ANSIC prototype: int32_t TimeDateToUnixSeconds(TIMEREC *time, DATEREC *date, int8_t offset_hours)

    • date: Pointer to DATEREC - Pointer to DATEREC with the date to convert
    • offset_hours:int8_t - Optional number of offset hours to adjust for the time zone. Use 0 for no adjustment.
    • Return value:int32_t - number of seconds after given base month/year
