Component minIni
minIni is a programmer’s library to read and write 'ini' files in embedded systems.
Component Level: High
Category: SW-User Components
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • minIni Version - Identifies the minIni version
  • SDK - Link to SDK used
  • Utility - Interface to utility string frunctions
  • Portable strnicmp() - If set to yes, it enables PORTABLE_STRNICMP in the configuration header file. With this, the minIni.c implements the strnicmp() function .
  • Use Real - If set to yes, defines INI_REAL in the configuration header file. If enabled, then the function ini_getf() is available to read floating point values.
  • Read Only - If set to yes, defines INI_READONLY in the configuration header file. Then access to the ini file is read only.
  • No Debug - If set to yes, then it defines NDEBUG in the configuration header file, so assert() calls are not performed.
  • Global Buffer - If set to 'yes', the module will use a global buffer instead a local buffer on the stack. This reduces stack usage for large buffer sizes, but makes things non-reentrant.
  • Shell - Optional interface to the shell
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Shell - Optional interface to the shell
  • FatFS - FatFS configuration items
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Buffer Size - Configures the macro INI_BUFFERSIZE which configures the maximum line length adn maxium path length.
    • FatFS - Link to FatFS (FAT File System) component.
  • Source Folders - Configures the source folder names
    • Source Folder - Folder name where to store the source files. Setting can be empty.
    • Config Folder - Folder name where to store the configuration source files. Setting can be empty.
